### 2. Strategy Interface Design

```go
// Strategy Interface
type Strategy interface {
    // Called on each new bar - main logic for generating signals
    OnBar(bar HistoricalPrice, portfolio Portfolio) Signal

    // Called when a signal is triggered
    OnSignal(signal Signal, portfolio Portfolio) *Order

    // Called when a trade is executed
    OnTrade(trade Trade, portfolio Portfolio)

    // Get strategy parameters
    GetParameters() map[string]interface{}

    // Get strategy name
    GetName() string
}

// Signal types
type Signal struct {
    Type       string    // BUY, SELL, HOLD
    Strength   float64   // 0-100 confidence
    Price      float64   // Entry price suggestion
    StopLoss   float64
    TakeProfit float64
}

// Order types
type Order struct {
    Type          string  // MARKET, LIMIT, STOP
    Side          string  // BUY, SELL
    Quantity      float64
    EntryPrice    float64
    StopLoss      float64
    TakeProfit    float64
}
```

#### Example Strategy Implementations

- **Moving Average Crossover Strategy**: Buy when fast MA crosses above slow MA, Sell when fast MA crosses below slow MA
- **RSI Strategy**: Buy when RSI < 30 (oversold), Sell when RSI > 70 (overbought)
- **MACD Strategy**: Buy when MACD line crosses above signal line, Sell when MACD line crosses below signal line

### 3. Performance Metrics Formulas

| Metric | Formula | Description |
|--------|---------|-------------|
| **Total Return** | `(Final Equity - Initial Equity) / Initial Equity × 100%` | Overall profit/loss percentage |
| **Annualized Return** | `(1 + Total Return)^(252/Trading Days) - 1` | Annualized performance |
| **Sharpe Ratio** | `(Rp - Rf) / σp` | Risk-adjusted return (Rf = risk-free rate) |
| **Sortino Ratio** | `(Rp - Rf) / σd` | Downside risk-adjusted return |
| **Max Drawdown** | `max(Peak - Valley) / Peak × 100%` | Largest peak-to-trough decline |
| **Max Drawdown Duration** | `max consecutive days in drawdown` | Longest drawdown period |
| **Win Rate** | `Winning Trades / Total Trades × 100%` | Percentage of profitable trades |
| **Profit Factor** | `Gross Profit / Gross Loss` | Ratio of wins to losses |
| **Average Win** | `Total Profit / Winning Trades` | Average profit per winning trade |
| **Average Loss** | `Total Loss / Losing Trades` | Average loss per losing trade |
| **Expectancy** | `(Win Rate × Avg Win) - ((1 - Win Rate) × Avg Loss)` | Expected value per trade |
| **Calmar Ratio** | `Annualized Return / Max Drawdown` | Return per unit of drawdown risk |

### 4. Walk-Forward Analysis

#### Concept
Train on historical period A, test on period B to avoid overfitting.

```
Timeline: [---Training---][---Testing---][---Training---][---Testing---]
          Jan-Mar 2024    Apr 2024      Apr-Jun 2024    Jul 2024
```

```go
type WalkForwardConfig struct {
    TrainingPeriodDays  int     // e.g., 90 days
    TestingPeriodDays   int     // e.g., 30 days
    StepForwardDays    int     // e.g., 7 days (rolling window)
    MinTrainingData    int     // minimum bars required
}

type WalkForwardResult struct {
    Fold                int
    TrainStart           time.Time
    TrainEnd             time.Time
    TestStart            time.Time
    TestEnd              time.Time
    TrainMetrics         PerformanceMetrics
    TestMetrics          PerformanceMetrics
    CoefficientOfVariation float64 // Stability measure
}
```
